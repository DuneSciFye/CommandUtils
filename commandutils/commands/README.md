# Commands

{% hint style="info" %}
Arguments enclosed in \<arrows> mean required, and arguments enclosed in \[\<brackets>] mean optional. When a command lists more than one **Usage** line, each line is a separate valid form of the command.
{% endhint %}

Every command can be individually enabled, disabled, renamed, or aliased in [config.yml](../config.md). Commands are designed to be run from command blocks, datapacks, other plugins (such as ExecutableItems/ExecutableBlocks), and the console.

Use the sidebar to browse the full command list.
